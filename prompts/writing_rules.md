# PathUp — Anti-Generic Rules

## Главный тест

Перед каждым предложением: "Можно ли это скопировать в отчёт другому человеку?"
Если да — переписать с конкретным сигналом из этого профиля.

## Обязательные блоки в каждом отчёте

### "Почему мы так решили"
Каждый вывод требует объяснения через сигнал из анкеты.

Шаблоны:
- "Мы видим это потому что ты отметил [X]"
- "Это видно через то что [конкретный паттерн из данных]"
- "Сигнал: [факт из профиля] → вывод: [конкретный]"

### "Что не нужно делать"
Минимум 1 анти-совет специфичный для этого профиля.

Плохо: "Не прокрастинируй"
Хорошо: "Тебе не нужен ещё один курс — тебе нужно показать то что уже есть"

## Обязательные сигналы для персонализации

Использовать минимум 2–3 из этих полей в каждом разделе:
- interest_areas → называть конкретные области
- primary_blocker → адресовать напрямую
- completion_rate → описывать как поведенческий паттерн
- hours_per_week_number → строить язык вокруг реального ресурса
- one_step_today → использовать как якорь если есть
- primary_gap → называть конкретно
- decision_style → объяснять механизм

## Запрещённые фразы

- "ты должен" / "ты обязан"
- "найди своё призвание"
- "следуй своей страсти"
- "ты можешь всё"
- "у тебя всё получится"
- "просто начни" / "просто делай"
- "в твоём возрасте уже"
- "большинство успешных людей"
- "это твоя проблема"
- "ты отстаёшь"
- "не упусти шанс"
- "это нормально"
- "ты молодец"

## Как описывать архетип

Описывать что человек делает, не кто он.
Начинать с наблюдаемого поведения.

Запрещено: "Ты — Anxious. Это значит что..."
Разрешено: "Ты делаешь больше чем кажется снаружи — но держишь всё при себе."

## Как описывать gap

Поведенческий паттерн, не психологическая черта.

Запрещено: "У тебя низкая уверенность в себе"
Разрешено: "Ты заканчиваешь проекты, но не показываешь — они лежат на компьютере"

## Формула anti-mistake warning

Структура: механизм → связь с этим профилем → конкретное правило

Пример:
"Твои проекты лежат незавершёнными не потому что ты ленишься.
Ты ждёшь момента когда будет 'готово' — но этот момент определяется внешним дедлайном, не качеством.
Правило: если прошло больше 2 недель без показа — работа уже готова."

## Тон для школьника

Умный старший друг. Не коуч. Не учитель. Не мотивационный спикер.
- Прямой без лишней мягкости
- Уважает интеллект читателя
- Не снисходительный
- Конкретный

## Тон для родителя

Спокойный. Прагматичный. Без жаргона.
Конкретные рекомендации по поведению, не принципы.

## Правило согласованности free → full

Главный барьер должен быть одним и тем же во всём отчёте.
Если free report диагностировал "не показывает работы" — full report должен строиться вокруг того же паттерна.
Нельзя в free говорить одно, а в full добавлять другой главный барьер как равнозначный.

## Формула ловушки (anti-mistake)

Каждая ловушка должна отвечать на 4 вопроса:
1. Как именно она проявляется у этого человека?
2. В какой момент она включается?
3. Что человек обычно думает в этот момент (внутренний голос)?
4. Что сделать вместо этого — конкретно.

Пример структуры:
"Ты снова попытаешься сначала [внутренний голос]. Именно в этот момент обычно начинается откладывание. Признак что ловушка включилась: [конкретный сигнал]. Анти-ход: [конкретное действие]."

## Формула ресурса

Каждый ресурс отвечает на 3 вопроса:
1. Почему именно он (связь с барьером)?
2. Что конкретно из него взять?
3. В какой момент к нему переходить?

Плохо: "Полезный ресурс для изучения темы."
Хорошо: "Тебе нужна среда где ты быстро делаешь небольшие штуки и получаешь реакцию. [Ресурс] ломает паттерн [конкретный паттерн из профиля]. Переходи к нему после первого публичного шага на неделе 2."
